# AntiSpam Moriarty Bot — v7

Антиспам + автоответы + панель в ЛС + модерация + приветствие + привлечение людей + выдача прав.

## Возможности

**Антиспам:** invite-ссылки, сокращатели, спам-домены, фильтр стоп-слов (чёрный список — сообщения удаляются), антифлуд (мут), бан при входе по спам-имени.

**Автоответы:** слово в чат → бот отвечает заданным текстом (пример `банан → 300 руб`).

**Модерация (в группе, для админов чата):**
`/ban` `/unban` `/kick` · `/mute [время]` `/unmute` · `/warn` `/unwarn` `/warns`
Цель: ответом на сообщение, либо `@username` / id. Время: `30m`, `2h`, `1d`. Предупреждения копятся до лимита, затем авто-наказание (мут или бан).

**Привлечение и постинг:**
- `/invite` — бот выдаёт ссылку-приглашение в группу (нужно право «приглашать пользователей»)
- `/zazyvala` — публикует в группе кнопку «Пригласить друга»: по тапу открывается окно «поделиться» с готовой ссылкой, участник выбирает друзей и отправляет. Текст редактируется в панели «Промо/Рассылка»
- `/all [текст]` — **призыв**: бот отмечает (тегает) всех активных участников, они получают уведомление. Без спама: одним вызовом, пачками
- `/anreg` — выйти из призывов, `/reg` — вернуться (команды для любого участника)
- **Постинг**: в панели «Промо/Рассылка» → «Запостить в группу» — выбираешь группу из списка и публикуешь в неё сообщение, не заходя в чат
- `/say текст` — бот публикует объявление в группе
- `/broadcast текст` — рассылка во все группы бота (в ЛС)
- **Авто-промо по таймеру** — бот сам постит сообщение во все группы через заданный интервал (панель → «Промо/Рассылка»)

**Приветствие новичков:** включается в панели, текст с `{name}` и `{chat}`.

**Выдача прав (только главный владелец):**
- `/grant` — дать человеку доступ к управлению ботом (ответом на его сообщение или `@username`/id)
- `/revoke` — забрать · `/managers` — список
Менеджеры получают панель и модерацию; выдавать/забирать права может только главный владелец (`ADMIN_IDS`).

**🛡 Защита от сноса (анти-снос):** бот следит за массовыми банами и сразу пишет тебе в ЛС, если: идёт массбан, бота сняли с админки/удалили, назначили нового админа, сменили название/фото группы. При массбане может сам забанить нарушителя. Настройки — в панели «Анти-снос». _Ограничение Telegram: создателя группы не может тронуть никто, даже бот._

**Правила и ссылка (для участников, в группе):**
- `/group` (или `/rules`) — показать правила группы
- `/link` — ссылка на группу
Текст правил редактируется в панели «Правила» или командой `/setrules`.

**Панель в ЛС:** все настройки на кнопках — функции, антифлуд, модерация, приветствие, промо/рассылка, доступ, ключевые слова, стоп-слова, домены.

## Деплой / обновление

**Обновить работающего бота:** если у тебя уже стоит v4, изменился только `bot.py` — залей его на GitHub (Add file → Upload files → перетащи `bot.py`, он заменит старый → Commit). Railway передеплоит сам. `requirements.txt` менять не нужно — он тот же.

_Если ставишь впервые или с версии ниже v4_ — залей и `bot.py`, и `requirements.txt` (в нём планировщик для авто-промо).

**С нуля:** репозиторий на GitHub → Railway (New Project → Deploy from GitHub repo) → переменная `BOT_TOKEN` → дать боту в группе права администратора: удалять сообщения, блокировать, ограничивать участников, **приглашать пользователей** (для ссылок).

## ⚠️ Сохранение настроек после передеплоя

Файловая система Railway сбрасывается при пересборке. Чтобы настройки, предупреждения и список групп не слетали: добавь в Railway **Volume** на путь `/data` и переменную `DATA_DIR=/data`.

## Доступ к сообщениям и несколько групп

Бот должен быть **администратором** в каждой группе (админы видят все сообщения и могут удалять/банить). Дополнительно у @BotFather можно отключить режим приватности, чтобы бот видел сообщения и до выдачи прав: @BotFather → /mybots → выбрать бота → Bot Settings → Group Privacy → Turn off, затем удалить бота из группы и добавить заново.

Бот работает в **нескольких группах одновременно**. Предупреждения, ссылки, списки участников и т.п. считаются отдельно по каждому чату. Настройки и правила — общие для всех групп.

## Безопасность

Токен — только в Variables на Railway. Главный доступ — у ID из `ADMIN_IDS`; остальным права выдаёшь через `/grant` и в любой момент забираешь.
